Installing the OpenShift Pipelines Operator

OpenShift Pipelines can be installed by using the OpenShift Pipelines Operator indexed in the OpenShift OperatorHub. The procedure for installing the OpenShift Pipelines Operator entails first creating a subscription and then creating the relevant Custom Resources and Custom Resoure Definitions (CRDs).

Prerequisites

  • An OpenShift 4 cluster is required for installation. Visit try.openshift.com for information on setting up an OpenShift 4 cluster.

Installing the Pipelines Operator Subscription

Procedure

  1. In the OpenShift Web Console, navigate to Home → Projects. Click Create Project.

  2. In the Create Project dialog box, type pipelines as the name of the project and click Create.

  3. Navigate to Catalog → OperatorHub.

  4. Use the Filter by Keyword box to facilitate the search of the OpenShift Pipelines Operator in the catalog. Click on the OpenShift Pipelines Operator tile.

  5. The OpenShift Pipelines Operator screen providing a brief description of the operator is displayed. Click Install.

  6. On the Create the Operator Subscription screen:

    1. Select a Specific namespace on the cluster, in the Installation Mode section,

    2. Select pipelines from the namespace drop-down list.

    3. Click Subscribe.

  7. Confirm the subscription for the installation operator, by viewing the Subscription Overview. Observe as the UPGRADE STATUS goes from 0 installing to 1 installed.

Creating Custom Resources for the OpenShift Pipelines Operator

Prerequisites

  • A subscription to the OpenShift Pipelines Operator is required. For more information, see Installing the OpenShift Pipelines Operator.

Procedure

  1. Click the 1 installed link.

  2. Click Create New to create an instance of the Openshift Pipelines install CRD.

  3. On the OpenShift Pipelines Install Custom Resources editor screen, ensure that the displayed namespace is pipelines. Click Create.

    The editor is pre-populated with definitions. Configure the file by manually entering YAML or JSON definitions into the editor.

  4. OpenShift Pipelines is now installed. In the OpenShift Pipelines Install tab, click on pipelines-install to view the install status.

  5. On the pipelines-install screen, select the Overview tab to view the Openshift Pipelines resources installed by the operator.

  6. Check on OpenShift Pipelines Operator by navigating to Workloads → Pods.
